Welcome!

Breakpoint is a new, open source code evaluation platform. This means that you can run code against our infrastructure in various ways in order to demonstrate how code works in a large number of languages.

We provide a simple-to-use JSON interface, which allows you to throw code at our servers and get a result almost instantly. Our platform is distributed, meaning that we can easily add servers to always keep Breakpoint as fast as possible.

How it works

We use a unique set of technologies to provide a stable, scalable platform that is always ready when you are.

At the top of our platform is our officially supported API clients. We offer officially supported clients in Clojure, Scala, Ruby, Python, JavaScript and PHP. But don't fear, making clients for other languages is easy!

The next level of the platform is what the API client connects to: Our web tier. The web stack is a simple Play! 2.0 application, written in Scala. When a JSON request hits the web application, it does some validity checking to ensure that you're a valid user and that the request is sane. Afterwards, it transforms the request into a Scala object, and sends it off to our Akka environment.

After the request hits Akka, it gets forwarded to the Breakpoint Compilation System, or "BCS." BCS creates an SELinux Sandbox, which is unique to the evaluation being done. BCS tells the sandbox how to process the request, and waits for its response. If it receives a response within the preset time limit, it sends another Scala object, (an EvaluationResponse() object), as a message back up to Akka, which forwards it to the web tier, and finally back to the client. If, however, no response is received, we kill the sandbox and send a EvaluationTimeout() message back to Akka. The web tier handles this as an error condition and informs the user of the timeout.

How we're different

Well, for one, we exist. ;)

There's not a whole lot of code evaluation platforms out there, and certainly no open source ones. But the technology exists to make a really nice platform, and we've decided to harness the power of it to create something we're proud of.

Breakpoint sets out to be a very community-centric, fast-paced project. With this comes challenges. We need to work out the best technology to use, get volunteers to help with upkeep, and be able to trust said volunteers.

We provide a fast, unique service for free. We allow anyone to contribute to it, fork it for themselves, and in general, make it better. We allow you to choose which architecture your code runs on, while providing unique ways to access the result (see the list of official API clients above).
